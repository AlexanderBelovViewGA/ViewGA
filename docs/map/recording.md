# Процесс записи и сбора данных (Capture Data)
---

После настройки параметров в окне **Tracking Map** начинается основной этап формирования базы снимков для будущей карты.

## Запуск и логика записи
---
Для начала процесса нажмите кнопку **Capture Data**. 

*   **Автоматизация**: Приложение начинает записывать снимки, на основе которых будет строиться 3D-модель. Алгоритм в реальном времени отслеживает параллакс, сохраняя кадры автоматически при достижении нужного смещения.
*   **Скорость работы**: Благодаря сверхкороткой экспозиции камер (**1–2 мс**), точки считываются с высокой точностью даже при резких движениях. Это позволяет проводить съемку в быстром темпе, буквально «на бегу».

>На текущий момент финальная сборка карты выполняется во внешнем специализированном ПО. Однако в разработке находится собственный алгоритм ViewGA, который позволит визуализировать построение 3D-облака точек прямо в интерфейсе системы.

## Завершение сессии
---
Съемка ведется по **кластерам** (группам меток на определенном участке: пол, стены или потолок). После того как вы отсняли весь целевой участок и набрали достаточное количество ракурсов, повторно нажмите **Capture Data** для остановки записи.

## Структура выходных данных
---
Все материалы сохраняются локально. Путь к данным по умолчанию:
`C:\CGF\viewga_4.2\data\maps\*имя проекта*\*дата создания*\`

### Содержимое папки:

1.  **Снимки**: Набор кадров, зафиксированных системой.
2.  **Технические данные**: Служебные файлы, используемые разработчиками для анализа.
3.  **Файл разметки (Agisoft Metashape)**: Ключевой файл для последующей обработки.

### Спецификация файла разметки:

*   **Метаданные**: Разрешение и индексированные имена снимков.
*   **Индексация точек**: Список меток (бывшие "L"), каждой из которых присвоен уникальный индекс.
*   **Координатная привязка**: Детальная разметка, указывающая, какой маркер на каком снимке и в каких координатах (пикселях) был обнаружен.
